| 路径 | 组件 | 守卫 | 说明 |

| `/settings` | `SettingsPage` | auth | 需要登录 |

### 3.2 路由守卫实现

```typescript
// 守卫逻辑（在 App.tsx 中实现）
function AuthGuard({ children }: { children: ReactNode }) {
  const { user, isLoading } = useAuth()

  if (isLoading) return <LoadingSkeleton />
  if (!user) return <Navigate to="/login" replace />

  return <>{children}</>
}

function AdminGuard({ children }: { children: ReactNode }) {
  const { user } = useAuth()

  if (!user?.isAdmin) return <Navigate to="/dashboard" replace />

  return <>{children}</>
}

function GuestGuard({ children }: { children: ReactNode }) {
  const { user } = useAuth()

  if (user) return <Navigate to="/dashboard" replace />

  return <>{children}</>
}
```

### 3.3 路由层级

```
<BrowserRouter>
  <ErrorBoundary>
    <Routes>
      {/* 公开路由 — 无 Layout */}
      <Route path="/login" element={<GuestGuard><LoginPage /></GuestGuard>} />
      <Route path="/" element={<GuestGuard><GuestPage /></GuestGuard>} />

      {/* 认证路由 — 有 Layout */}
      <Route element={<AuthGuard><Layout /></AuthGuard>}>
        <Route path="/dashboard" element={<DashboardPage />} />
        <Route path="/learn" element={<LearnPage />} />
        <Route path="/progress" element={<ProgressPage />} />
        <Route path="/settings" element={<SettingsPage />} />
        <Route path="/admin" element={<AdminGuard><AdminPage /></AdminGuard>} />
      </Route>

      {/* 404 */}
      <Route path="*" element={<Navigate to="/" replace />} />
    </Routes>
  </ErrorBoundary>
</BrowserRouter>
```

> **布局嵌套**：认证路由使用 `<Layout>` 作为父路由的 `element`，内部使用 `<Outlet />` 渲染子路由。
> 当前 DEMO 使用 `{children}` 方式（`Layout` 包裹子组件），后续可切换到 `<Outlet>` 模式。

## 4. 状态架构

### 4.1 全局状态（Context）

| Context | 存储内容 | 更新时机 | 消费组件 |
|---------|----------|----------|----------|
| `AuthContext` | `user`, `token`, `isLoading` | 应用加载、登录、登出 | App 守卫、Layout 顶栏（含登出按钮） |
| `SettingsContext` | 用户设置（newWordsPerSession等） | 应用加载、设置页保存 | LearnPage、Layout |

#### AuthContext 设计

```typescript
// context/AuthContext.tsx
interface AuthState {
  user: User | null
  token: string | null
  isLoading: boolean        // 初始加载中
}

interface AuthContextValue extends AuthState {
  login: (username: string, password: string) => Promise<void>
  register: (username: string, password: string) => Promise<void>
  logout: () => void
  refreshUser: () => Promise<void>   // 刷新用户信息
}
```

**初始化流程**：

1. 应用挂载 → `AuthContext` 检查 `localStorage` 中是否有 token
2. 有 token → 调用 `GET /api/auth/me` 验证 token 有效性
3. 有效 → 设置 `user`，`isLoading = false`
4. 无效/无 token → `isLoading = false`，视为未登录
5. Token 过期 → 清除 localStorage，跳转登录页

#### SettingsContext 设计

```typescript
// context/SettingsContext.tsx
interface SettingsContextValue {
  settings: UserSettings
  updateSettings: (partial: Partial<UserSettings>) => Promise<void>
  isLoading: boolean
}
```

**初始化**：从 `GET /api/auth/me` 返回的 `settings` 字段中获取，无需单独请求。
**更新**：调用 `PUT /api/settings`，成功后更新本地状态。

### 4.2 页面级状态

每个页面组件管理自己的局部状态，不提升到全局，除非被 2+ 页面共享。

| 页面 | 状态 | 类型 |
|------|------|------|
| `GuestPage` | `stats: PublicStats \| null` | `useState` |
| `LoginPage` | `tab, form fields, errors, loading` | `useState` |
| `DashboardPage` | `tasks, progress, heatmap, distribution` | `useState`(每个独立) |
| `LearnPage` | `phase, currentIdx, selected, scores, wordQueue` | `useReducer`（推荐） |
| `ProgressPage` | `rounds, distribution, heatmap` | `useState` |
| `SettingsPage` | `settings form` (受控) | `useState` + Context |
| `AdminPage` | `users[], search, loading` | `useState` |

### 4.3 LearnPage 状态设计（关键页面）

使用 `useReducer` 管理学习页的复杂状态转移：

```typescript
// LearnPage 状态
interface LearnState {
  phase: 'mode_select' | 'review' | 'new_words' | 'final_test' | 'complete'
  wordMode: 'random' | 'alpha' | 'A' | 'B' | ... | 'Z' | null
  wordScope: string              // 本轮单词范围: 'all'(默认) 或单字母 'A'~'Z'
  sessionId: number | null
  currentIndex: number
  selectedChoice: number | null
  answerLocked: boolean         // 选择后锁定，展示反馈
  scores: boolean[]             // 本轮答题记录
  wordQueue: WordItem[]         // 当前队列（复习/新词/总测）
  retestQueue: number[]         // 答错的 wordId 列表
  round: number
}

type LearnAction =
  | { type: 'SELECT_MODE'; mode: 'random' | 'alpha' }
  | { type: 'START_SESSION'; sessionId: number; round: number }
  | { type: 'LOAD_WORDS'; words: WordItem[]; phase: LearnState['phase'] }
  | { type: 'CHOOSE_ANSWER'; choiceIndex: number }
  | { type: 'NEXT_QUESTION' }
  | { type: 'PHASE_TRANSITION'; nextPhase: LearnState['phase'] }
  | { type: 'COMPLETE_SESSION'; result: CompleteSessionResponse }
```

**状态转移图**：

```
mode_select ──[选择模式 + 点击开始]──▶ review
review ──[答完最后一题]──▶ new_words
new_words ──[答完最后一题]──▶ final_test
final_test ──[全部正确]──▶ complete
final_test ──[有错]──▶ final_test (retest 循环)
complete ──[点击返回]──▶ (导航到 /dashboard)
```

## 5. API 客户端层

### 5.1 API 封装（`lib/api.ts`）

```typescript
// 统一 API 客户端
const BASE_URL = '/api'

interface ApiOptions {
  method?: 'GET' | 'POST' | 'PUT' | 'DELETE'
  body?: unknown
  query?: Record<string, string | number | undefined>
  authenticated?: boolean   // 默认 true
}

async function apiClient<T>(endpoint: string, options: ApiOptions = {}): Promise<T> {
  const { method = 'GET', body, query, authenticated = true } = options

  // 构建 URL
  let url = `${BASE_URL}${endpoint}`
  if (query) {
    const params = new URLSearchParams()
    Object.entries(query).forEach(([k, v]) => { if (v !== undefined) params.set(k, String(v)) })
    const qs = params.toString()
    if (qs) url += `?${qs}`
  }

  // 构建 headers
  const headers: Record<string, string> = { 'Content-Type': 'application/json' }
  if (authenticated) {
    const token = getToken()    // 从 localStorage 获取
    if (token) headers['Authorization'] = `Bearer ${token}`
  }

  // 发起请求
  const res = await fetch(url, {
    method,
    headers,
    body: body ? JSON.stringify(body) : undefined,
  })

  // 错误统一处理
  if (!res.ok) {
    const errBody = await res.json().catch(() => ({}))
    const error = errBody.error || { code: 'UNKNOWN', message: `HTTP ${res.status}` }

    // Token 过期 → 自动登出
    if (error.code === 'INVALID_TOKEN') {
      clearToken()
      window.location.href = '/login'
      throw error
    }

    throw error
  }

  const json = await res.json()
  return json.data as T
}

// 导出便捷方法
export const api = {
  get: <T>(endpoint: string, query?: Record<string, string | number | undefined>) =>
    apiClient<T>(endpoint, { query }),
  post: <T>(endpoint: string, body?: unknown) =>
    apiClient<T>(endpoint, { method: 'POST', body }),
  put: <T>(endpoint: string, body?: unknown) =>
    apiClient<T>(endpoint, { method: 'PUT', body }),
  delete: <T>(endpoint: string) =>
    apiClient<T>(endpoint, { method: 'DELETE' }),
}
```

export function getToken(): string | null {
  return localStorage.getItem(TOKEN_KEY)
}

export function setToken(token: string): void {
  localStorage.setItem(TOKEN_KEY, token)
}

  localStorage.removeItem(TOKEN_KEY)
}

### 5.3 Loading / Error Hook（`hooks/useApi.ts`）

```typescript
function useApi<T>() {
  const [data, setData] = useState<T | null>(null)
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState<ApiError | null>(null)

  const execute = useCallback(async (fetcher: () => Promise<T>) => {
    setLoading(true)
    setError(null)
    try {
      const result = await fetcher()
      setData(result)
      return result
    } catch (e) {
      setError(e as ApiError)
      throw e
    } finally {
      setLoading(false)
    }
  }, [])

  return { data, loading, error, execute }
}
```

## 10. 关键实现细节

### 10.1 干扰项生成算法

干扰项来源优先级（由高到低）：
1. 全量字典池 — 页面初始化时通过 `GET /learn/distractors?count=50` 预取
2. 当前 wordQueue — 学习队列中的其他词
3. `（干扰项）` — 极端兜底

```typescript
// lib/shuffle.ts
function generateChoices(
  correctWord: WordItem,
  pool: WordItem[],                    // 当前 wordQueue（补足用）
  count: number = 4,
  externalDistractors?: string[]       // 全量字典池（优先使用）
): ChoiceItem[] {
  // 1. 优先从全量字典池中取干扰项
  const seen = new Set<string>([correctWord.chinese])
  let candidates: string[] = []
  if (externalDistractors) {
    candidates = externalDistractors.filter(c => !seen.has(c))
    candidates.forEach(c => seen.add(c))
  }

  // 2. 不足时从当前 wordQueue 补
  if (candidates.length < count - 1) {
    const extras = pool
      .filter(w => w.wordId !== correctWord.wordId && !seen.has(w.chinese))
      .map(w => w.chinese)
    candidates = [...candidates, ...extras]
  }

  const distractors = candidates
    .sort(() => Math.random() - 0.5)
    .slice(0, count - 1)
    .map(text => ({ text, correct: false }))

  // 3. 兜底
  while (distractors.length < count - 1) {
    distractors.push({ text: '（干扰项）', correct: false })
  }

  // 4. 组装 + 乱序
  const choices: ChoiceItem[] = [
    { text: correctWord.chinese, correct: true },
    ...distractors,
  ]
  return shuffleArray(choices)
}
```

### 10.2 答题模式切换

由 `determineAnswerType(answerCount, spellingMode)` 决定每题的答题类型：

| `spellingMode` | `answerCount` | 结果 |
|:---|:---|:---|
| `false` | 任意 | `choice` |
| `true` | `< 3` | `choice` |
| `true` | `>= 3` | `choice` / `spelling` 各 50% 随机 |

```typescript
function determineAnswerType(answerCount: number, spellingMode: boolean): 'choice' | 'spelling' {
  if (!spellingMode) return 'choice'
  if (answerCount < 3) return 'choice'
  return Math.random() < 0.5 ? 'choice' : 'spelling'
}
```

当前不按单词维度区分（`answerCount` 是场次内全局累加值），也不按 proficiency 做特殊路由。

### 10.3 学习阶段自动过渡

```typescript
// 在 handleNext() 中的过渡逻辑
function handlePhaseTransition(
  phase: Phase,
  scores: boolean[],
  retestQueue: number[],
  wordQueue: WordItem[],
  correctCount: number,
  totalCount: number,
): Phase {
  if (phase === 'review' || phase === 'new_words') {
    return phase === 'review' ? 'new_words' : 'final_test'
  }

  if (phase === 'final_test') {
    // 总测有错 → 进入 retest 循环
    if (scores.slice(-wordQueue.length).some(s => !s)) {
      // 重置为 final_test，队列替换为重考词
      return 'final_test'   // 不改变 phase，但切换 wordQueue
    }
    return 'complete'
  }

  return phase
}
```

}

function ProgressBar({ value, max }: { value: number; max: number }) {
  const pct = Math.round((value / max) * 100)

  return (
    <div className="w-full bg-gray-100 rounded-full h-3">
      <div
        className={`h-3 rounded-full transition-all ${
          isPerfect(pct)
            ? 'bg-gradient-to-r from-amber-400 via-yellow-400 to-amber-500 shadow-lg shadow-amber-200 animate-pulse'
            : 'bg-gradient-to-r from-blue-500 to-indigo-500'
        }`}
        style={{ width: `${pct}%` }}
      />
    </div>
  )
}

function PercentDisplay({ value }: { value: number }) {
  if (isPerfect(value)) {
    return (
      <span className="text-2xl font-bold text-amber-500 drop-shadow-lg">
        👑 100%
      </span>
    )
  }
  return <span className="text-2xl font-bold text-blue-600">{value}%</span>
}
```

function QuizCard({ word, answerType, onAnswer }: QuizCardProps) {
  if (answerType === 'choice') {

function SpellingQuiz({ word, onAnswer }: { word: WordItem; onAnswer: (correct: boolean, ms: number) => void }) {
  const [input, setInput] = useState('')
  const startTime = useRef(Date.now())

  const handleSubmit = () => {
    const correct = input.trim() === word.word   // 大小写敏感
    const ms = Date.now() - startTime.current
    onAnswer(correct, ms)
  }

  return (
    <div>
      <p className="text-lg font-medium text-gray-700 mb-2">{word.chinese}</p>
      {settings.firstLetterHint && (
        <p className="text-sm text-gray-400 mb-1">{word.phonetic}</p>
      )}
      <p className="text-sm text-gray-400 mb-4">首字母: {word.word[0]}{'_ '.repeat(word.word.length - 1)}</p>
      <input
        type="text"
        value={input}
        onChange={e => setInput(e.target.value)}
        onKeyDown={e => e.key === 'Enter' && handleSubmit()}
        className="w-full px-4 py-3 text-lg text-center border rounded-xl focus:outline-none focus:ring-2 focus:ring-blue-500/20"
        placeholder="输入英文单词"
        autoFocus
      />
      <button onClick={handleSubmit} disabled={!input.trim()}
        className="mt-4 w-full py-3 bg-blue-600 text-white rounded-xl disabled:bg-gray-200">
        确认
      </button>
    </div>
  )
}
```

### 10.6 时间估算公式（设置页实时更新）

```typescript
function estimateTime(newWordsPerSession: number): number {
  const reviewTime   = 20 * 12           // ~20 复习词 × 12s
  const newWordTime  = newWordsPerSession * 30  // N 新词 × 30s
  const finalTestTime = 30 * 12          // ~30 总测词 × 12s
  const bufferTime   = 3 * 60            // 3 分钟缓冲
  return Math.round((reviewTime + newWordTime + finalTestTime + bufferTime) / 60)
}
```

### 10.8 错误处理策略

```
网络错误 (fetch 抛出)
  → Toast: "网络连接失败，请检查网络后重试"
  → 自动重试按钮

API 错误 (4xx/5xx)
  → 根据 error.code 显示对应中文消息
  → Token 过期 → 自动登出 + 跳转登录页
  → 403 → Toast: "没有操作权限"
  → 409 → 表单字段级错误提示

空数据
  → 展示空状态引导文案
  - Dashboard: "开始你的第一次学习吧！" + [开始学习] 按钮
  - Progress: "暂无学习数据，完成第一次学习后这里将展示你的进度"
  - Admin: "暂无用户"

加载态
  → 骨架屏（Skeleton）
  - 卡片列表: 3 个灰色脉冲卡片
  - 表格: 5 行灰色脉冲行
  - 图表: 灰色矩形
```
