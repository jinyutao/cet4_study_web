## 1. 设计原则

### 1.1 核心定位

学习工具类产品，强调 **清晰、专注、正向反馈**。避免游戏化过度设计，保持工具感的同时提供适当的情感激励。

### 1.2 设计语言

| 原则 | 说明 |
|------|------|
| **内容优先** | 单词卡片是页面绝对核心，所有装饰元素不干扰内容 |
| **渐进呈现** | 信息分层展示，避免一次性信息过载 |
| **即时反馈** | 每次答题立即显示正误，视觉反馈明确 |
| **一致性** | 跨页面保持相同的交互模式、色彩语义、间距系统 |
| **容错设计** | 不可逆操作（冻结、删除）需二次确认 |

### 1.3 色彩语义

| 用途 | 色值 | 出现场景 |
|------|------|----------|
| 主要操作 / 链接 | `blue-600 (#2563eb)` <span style="display:inline-block;width:14px;height:14px;border-radius:3px;background:#2563eb;vertical-align:middle;border:1px solid rgba(0,0,0,0.1);"></span> | 按钮、导航激活态、链接 |
| 成功 / 正确 | `emerald-500 (#10b981)` <span style="display:inline-block;width:14px;height:14px;border-radius:3px;background:#10b981;vertical-align:middle;border:1px solid rgba(0,0,0,0.1);"></span> | 答题正确、已掌握、完成 |
| 错误 / 警告 | `red-500 (#ef4444)` <span style="display:inline-block;width:14px;height:14px;border-radius:3px;background:#ef4444;vertical-align:middle;border:1px solid rgba(0,0,0,0.1);"></span> | 答题错误、已冻结、删除 |
| 次要文案 | `gray-400 (#9ca3af)` <span style="display:inline-block;width:14px;height:14px;border-radius:3px;background:#9ca3af;vertical-align:middle;border:1px solid rgba(0,0,0,0.1);"></span> | 辅助说明、时间戳、统计数字 |
| 背景底色 | `gray-50 (#f9fafb)` <span style="display:inline-block;width:14px;height:14px;border-radius:3px;background:#f9fafb;vertical-align:middle;border:1px solid rgba(0,0,0,0.1);"></span> | 页面背景、卡片背景 |
| 卡片表面 | `white (#ffffff)` <span style="display:inline-block;width:14px;height:14px;border-radius:3px;background:#ffffff;vertical-align:middle;border:1px solid rgba(0,0,0,0.1);"></span> | 内容卡片、表单区域 |

## 2. 设计系统

### 2.1 字体

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Noto Sans SC', sans-serif;
-webkit-font-smoothing: antialiased;
```

- 优先使用系统原生字体族，避免额外字体加载
- `Noto Sans SC` 提供中文字体回退
- 无衬线体保证屏幕可读性

### 2.2 字号梯度

| Token | 大小 | 用途 |
|-------|------|------|
| `text-xs` | 12px | 辅助信息、标签、统计数据 |
| `text-sm` | 14px | 正文、按钮文字、表单标签 |
| `text-base` | 16px | 主要正文 |
| `text-lg` | 18px | 标题、强调文字 |
| `text-xl` | 20px | 页面标题 |
| `text-2xl` | 24px | 大标题 |
| `text-3xl` | 30px | 单词展示（学习页） |
| `text-4xl` | 36px | 单词展示大号（学习页） |

### 2.3 间距系统

采用 4px 基准倍数：

| Token | 值 | 用途 |
|-------|-----|------|
| `gap-1` | 4px | 紧密排列的标签 |
| `gap-1.5` | 6px | 小元素间距 |
| `gap-2` | 8px | 图标+文字间距 |
| `gap-3` | 12px | 表单项间距、段落间距 |
| `gap-4` | 16px | 卡片内元素间距 |
| `gap-5` | 20px | 卡片内段落间距 |
| `gap-6` | 24px | 页面区块间距 |
| `p-4` | 16px | 卡片内边距 |
| `p-5` | 20px | 卡片内边距 |
| `p-6` | 24px | 页面边距 |
| `px-6` | 24px | 页面水平边距 |
| `py-6` | 24px | 页面垂直边距 |

### 2.4 圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `rounded-lg` | 8px | 按钮、输入框、标签 |
| `rounded-xl` | 12px | 卡片、大的按钮 |
| `rounded-2xl` | 16px | 大卡片、模态框 |

### 2.5 阴影

| Token | 用途 |
|-------|------|
| `shadow-sm` | 卡片默认阴影 |
| `shadow-md` | 选中态卡片、主要按钮 |
| `shadow-lg` | 模态框、浮动元素 |
| `shadow-xl` | 登录卡片 |

### 2.6 100% 掌握率特殊显示

掌握率达到 100% 时，在以下所有出现位置使用**庆祝态替代常规显示**：

| 常规态 | 100% 庆祝态 |
|--------|------------|
| 蓝色 `text-blue-600` | 金色 `text-amber-500` |
| 蓝色进度条 | 金色渐变 `from-amber-400 via-yellow-400 to-amber-500` |
| 常规 Emoji | 🎉 👑 🏆 交替使用 |
| 普通数字 | 加粗放大 + 发光文字效果 `text-shadow` |
| 静态标签 | 闪烁星星动画 ⭐ |

**触发规则**：
- 熟练度分布中 Lv5 占比 = 100%
- 单轮进度条达到 100%
- 学习结算正确率 = 100%
- Dashboard 总进度 = 100%

**视觉方案**：

```html
<!-- 进度条 100% 庆祝态 -->
<div class="h-3 rounded-full bg-gradient-to-r from-amber-400 via-yellow-400 to-amber-500 
            shadow-lg shadow-amber-200 animate-pulse">
</div>

<!-- 百分比数字 100% 庆祝态 -->
<span class="text-2xl font-bold text-amber-500 drop-shadow-lg">
  👑 100%
</span>

<!-- 结算页 100% 特殊动画环形 -->
<circle ... stroke="url(#goldGradient)" strokeDasharray="327 327" />
<defs>
  <linearGradient id="goldGradient" x1="0" y1="0" x2="1" y2="1">
    <stop offset="0%" stop-color="#f59e0b" />
    <stop offset="100%" stop-color="#d97706" />
  </linearGradient>
</defs>
```

## 3. 布局系统

### 3.1 整体结构

```
┌─────────────────────────────────────────────┐
│  Top bar (PC only)                          │
│  logo + 用户名 + 头像                         │
├──────────┬──────────────────────────────────┤
│ Sidebar  │  Main content area               │
│ (PC)     │  max-w-4xl mx-auto               │
│ 56px     │  px-4 py-6                       │
│          │                                  │
│ nav      │  ┌──────────────────────────┐    │
│ items    │  │  Page content            │    │
│          │  │                          │    │
│          │  └──────────────────────────┘    │
├──────────┴──────────────────────────────────┤
│  Bottom tab bar (mobile only)               │
└─────────────────────────────────────────────┘
```

### 3.2 布局断点

| 断点 | 宽度 | 导航形态 | 内容布局 |
|------|------|----------|----------|
| `md` | ≥768px | 顶部栏 + 左侧边栏 (224px) | 多栏网格 |
| `<md` | <768px | 底部固定 Tab Bar | 单栏堆叠 |

### 3.3 导航

**PC 端（≥768px）**
- 顶部栏：左侧 Logo + 标题，右侧用户名 + 圆形头像首字母 + 登出按钮
- 左侧边栏：5 个导航项，带 Emoji 图标，激活态蓝色背景高亮
- 登出按钮位于顶部栏右侧，点击后清除 token 并跳转登录页

**移动端（<768px）**
- 底部固定 Tab Bar：5 项均分，激活态蓝色，Emoji 图标在上方
- 顶栏隐藏，侧栏隐藏
- 内容区底部预留 `pb-20` 防止 Tab Bar 遮挡
- 登出入口在设置页面底部（「账号」区块内）

### 3.4 内容区

- `max-w-4xl mx-auto` — 最大宽度 896px，居中
- 内部使用 `space-y-6` 分隔区块
- 区块多采用白色卡片 (`bg-white rounded-2xl p-5 shadow-sm`)

## 4. 页面详解

### 4.1 访客页 `/` (GuestPage)

**目标**：未登录用户了解系统、注册转化

**区块结构**：
```
┌─────────────────────────────────────────────┐
│  Top bar: Logo + [登录] [注册]                │
├─────────────────────────────────────────────┤
│  Hero: 标题 + 描述 + CTA 按钮                 │
├─────────────────────────────────────────────┤
│  统计卡片: 5 个指标 (grid-cols-2 md:5)        │
├─────────────────────────────────────────────┤
│  学习榜 + 最近学习动态 (md:grid-cols-2)        │
├─────────────────────────────────────────────┤
│  核心功能: 6 特征卡片 (md:grid-cols-3)         │
├─────────────────────────────────────────────┤
│  Footer                                     │
└─────────────────────────────────────────────┘
```

**视觉**：渐变背景 (`bg-gradient-to-br from-blue-50 via-white to-indigo-50`)，卡片彩色分类（蓝/绿/紫/翠绿/琥珀）

**交互**：
- CTA 按钮 `免费注册开始学习 🚀` → 跳转 `/register`
- 登录/注册按钮 → 跳转对应页面

**数据**：
- 5 统计卡片：总用户、本月活跃、总词汇量、已掌握、学习次数
- 学习榜：Top 3 学习者（排名 + 用户名 + 掌握词数 + 熟练度）
- 动态：柱状图展示近 5 天学习量趋势

### 4.2 登录/注册页 `/login` (LoginPage)

**目标**：用户认证，登录或注册新账号

**区块结构**：
```
┌─────────────────────────────────────────────┐
│  Logo + 标题 + 副标题                         │
├─────────────────────────────────────────────┤
│  登录/注册 Tab 切换器                         │
│  ┌──────┬──────┐                             │
│  │ 登录  │ 注册  │   ← tab bar              │
│  └──────┴──────┘                             │
│  用户名 输入框                                │
│  密码 输入框                                  │
│  确认密码 输入框 (仅注册)                      │
│  提交按钮                                     │
│  底部切换链接                                  │
└─────────────────────────────────────────────┘

- 输入框 `bg-gray-50` 默认态，`border-blue-300` focus 态
- 提交按钮 `shadow-lg shadow-blue-200`

│  └─ 提醒时间: [time picker]                  │
├─────────────────────────────────────────────┤
│  👤 账号                                     │
│  ├─ 修改密码                                  │
│  ├─ 导出数据                                  │

│  └─ 重置本轮进度 (橙色警告)                   │
└─────────────────────────────────────────────┘
```

**交互细节**：
- 滑块实时显示当前值（右侧数字标签）
- 每日目标滑块下方实时估算学习时长
- 时间估算基于设计文档 §6.5 模型：
  - 复习 ~12s/词，新词 ~30s/词，总测 ~12s/词，缓冲 ~3min
- Toggle 开关组件：`role="switch"` + `aria-checked`，过渡动画
- 选项数三选一按钮组
- 提醒时间：原生 `<input type="time">`
- 账号操作为按钮容器（DEMO 阶段无实际功能）

## 5. 交互流程

### 5.1 答题核心交互

```
用户看到单词 + 音标
      │
      ▼
  阅读四个选项
      │
      ▼
  点击选择答案
      │
      ├── 正确 → 选项变绿色 + ✅ 回答正确
      └── 错误 → 用户选项变红色 + 高亮正确答案 + ❌ 正确答案: XX
      │
      ▼
  「下一题」按钮激活（之前禁用）
      │
      ▼
  点击下一题/进入下一步
```

### 5.2 模态确认

所有不可逆操作使用 `window.confirm()`：
- 重置密码: "确认重置用户 "XX" 的密码？"
- 冻结/解冻: "确认冻结用户 "XX"？冻结后用户将无法登录" / "确认解冻用户 "XX"？"
- 删除: "确认永久删除用户 "XX"？该用户的所有学习数据将一并清除，此操作不可恢复。"
- 设置/取消管理员: "确认设置为管理员权限给 "XX"？" / "确认取消管理员权限给 "XX"？"

### 5.3 页面过渡

- 学习页阶段切换：自动过渡（点击「下一步」后切换 Phase）
- 结算→首页：手动点击「返回首页」
- 登录注册 Tab 切换：清空错误状态

## 7. 响应式适配

### 7.1 断点策略

| 断点 | 宽度 | 对应设备 | 布局变化 |
|------|------|----------|----------|
| 默认 | <768px | 手机 | 单栏，底部 Tab，选项 1 列 |
| `sm` | ≥640px | 大屏手机 | 部分网格从 1 列变 2 列 |
| `md` | ≥768px | iPad | 顶栏 + 侧栏出现，导航从底部切到左侧 |
| `lg` | ≥1024px | PC | 双栏内容布局，全宽 |

### 7.2 各页面适配要点

| 页面 | 手机 (<768px) | PC (≥768px) |
|------|--------------|-------------|
| 访客页 | 统计卡片 2 列 | 5 列，特征 3 列 |
| 登录页 | 全宽卡片 | 居中 max-w-md |
| Dashboard | 单栏堆叠 | lg 双栏 |
| 学习页 | 选项 1 列 | 选项 2×2 |
| 统计页 | 单栏堆叠 | 轮次 2 列，概要 3 列 |
| 设置页 | 全宽 | max-w-2xl |
| 管理页 | 表格全显示 | 注册时间 `md:table-cell` |

### 7.3 移动 Tab Bar

- `fixed bottom-0 left-0 right-0` 固定底部
