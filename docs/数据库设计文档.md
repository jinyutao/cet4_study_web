通过 `docker-compose.test.yml` 覆层切换数据库：

```yaml

```

```
http://server:9098/test_db?db=cet4_test.db → 迷你测试库
```

## 4. 表定义

### 4.1 `words` — 词汇表

CET-4 大纲词汇，预填充，只读（应用层不修改）。

```sql
CREATE TABLE IF NOT EXISTS words (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  word        TEXT NOT NULL UNIQUE,
  phonetic    TEXT,
  pos         TEXT,
  chinese     TEXT NOT NULL,
  english_def TEXT
);
```

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| `id` | INTEGER | PK, AUTOINCREMENT | 自增主键 |
| `word` | TEXT | NOT NULL, UNIQUE | 单词本身，如 "abandon" |
| `phonetic` | TEXT | — | 音标，如 "[əˈbændən]" |
| `pos` | TEXT | — | 词性：`v`/`n`/`a`/`ad`/`prep`/`art`/`pron` |
| `chinese` | TEXT | NOT NULL | 中文释义，如 "丢弃；放弃，抛弃" |
| `english_def` | TEXT | — | 英文释义，可为 NULL |

**数据量**: 4517 行（源文件 4526 词，过滤掉含连字符/纯数字条目）

### 4.2 `users` — 用户表

```sql
CREATE TABLE IF NOT EXISTS users (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  username        TEXT NOT NULL UNIQUE,
  password_hash   TEXT NOT NULL,
  is_admin        INTEGER DEFAULT 0,
  frozen          INTEGER DEFAULT 0,
  created_at      TEXT DEFAULT (datetime('now'))
);
```

| 列 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `id` | INTEGER | — | 自增主键 |
| `username` | TEXT | — | 用户名，唯一 |
| `password_hash` | TEXT | — | bcrypt 哈希值，如 `$2b$10$...` |
| `is_admin` | INTEGER | `0` | 1=超级管理员（可管理所有用户） |
| `frozen` | INTEGER | `0` | 1=冻结（不可登录，数据保留） |
| `created_at` | TEXT | `datetime('now')` | ISO 8601 注册时间 |

### 4.3 `user_words` — 用户-词汇学习状态

核心表，记录每个用户对每个单词的学习进度。**复合主键** (user_id, word_id)。

```sql
CREATE TABLE IF NOT EXISTS user_words (
  user_id             INTEGER NOT NULL REFERENCES users(id),
  word_id             INTEGER NOT NULL REFERENCES words(id),
  ef                  REAL DEFAULT 2.5,
  interval_days       INTEGER DEFAULT 0,
  repetitions         INTEGER DEFAULT 0,
  proficiency         INTEGER DEFAULT 0,
  next_review         TEXT,
  total_correct       INTEGER DEFAULT 0,
  total_attempts      INTEGER DEFAULT 0,
  avg_response_time   REAL,
  round               INTEGER DEFAULT 1,
  consecutive_correct INTEGER DEFAULT 0,
  last_reviewed_at    TEXT,
  first_learned_at    TEXT,
  PRIMARY KEY (user_id, word_id)
);
```

| 列 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `user_id` | INTEGER | — | 用户 ID（FK → users.id） |
| `word_id` | INTEGER | — | 单词 ID（FK → words.id） |
| `ef` | REAL | `2.5` | SM-2 易度因子 [1.3, 3.0] |
| `interval_days` | INTEGER | `0` | SM-2 当前间隔天数 |
| `repetitions` | INTEGER | `0` | SM-2 连续正确次数 |
| `proficiency` | INTEGER | `0` | 综合熟练度 [0, 100] |
| `next_review` | TEXT | — | ISO 日期，下次复习时间 |
| `total_correct` | INTEGER | `0` | 历史答对总次数 |
| `total_attempts` | INTEGER | `0` | 历史答题总次数 |
| `avg_response_time` | REAL | — | 平均响应时间（毫秒） |
| `round` | INTEGER | `1` | 当前轮次 |
| `consecutive_correct` | INTEGER | `0` | 当前连续正确次数 |
| `last_reviewed_at` | TEXT | — | 上次复习时间 |
| `first_learned_at` | TEXT | — | 首次学习时间 |

### 4.4 `sessions` — 学习场次

```sql
  words_reviewed  INTEGER DEFAULT 0,
  words_passed    INTEGER DEFAULT 0,
  words_failed    INTEGER DEFAULT 0,
  duration_seconds INTEGER,
  round           INTEGER DEFAULT 1,
  status          TEXT DEFAULT 'active'
);
```

| 列 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `id` | INTEGER | — | 自增主键 |
| `user_id` | INTEGER | — | FK → users.id |
| `start_time` | TEXT | `datetime('now')` | 会话开始时间 |
| `end_time` | TEXT | — | 会话结束时间 |
| `words_reviewed` | INTEGER | `0` | 本次复习词数 |
| `words_passed` | INTEGER | `0` | 通过词数 |
| `words_failed` | INTEGER | `0` | 失败词数 |
| `duration_seconds` | INTEGER | — | 持续秒数 |

| `status` | TEXT | `'active'` | `active` / `completed` / `abandoned` |

## 5. 索引

```sql
-- user_words 查询加速
CREATE INDEX IF NOT EXISTS idx_user_words_next_review
  ON user_words(user_id, next_review);
  -- 用途: 查询某用户到期需要复习的词
  -- 覆盖: WHERE user_id=? AND next_review <= datetime('now')

CREATE INDEX IF NOT EXISTS idx_user_words_round
  ON user_words(user_id, round);
  -- 用途: 按轮次筛选学习记录

CREATE INDEX IF NOT EXISTS idx_user_words_proficiency
  ON user_words(user_id, proficiency);
  -- 用途: 查询熟练度分布、已掌握词数

-- review_logs 分析
CREATE INDEX IF NOT EXISTS idx_review_logs_user_word
  ON review_logs(user_id, word_id);
  -- 用途: 查某个词的历史答题记录

CREATE INDEX IF NOT EXISTS idx_review_logs_session
  ON review_logs(session_id);
  -- 用途: 按 session 聚合答题详情

CREATE INDEX IF NOT EXISTS idx_review_logs_created
  ON review_logs(user_id, created_at);
  -- 用途: 热力图、学习日历（按天聚合）

-- sessions
CREATE INDEX IF NOT EXISTS idx_sessions_user
  ON sessions(user_id, start_time);
  -- 用途: 用户的历史 session 列表
```

### 索引策略说明

| 查询模式 | 使用索引 | 说明 |
|----------|----------|------|
| 取到期复习词 | `idx_user_words_next_review` | 最频繁的查询，复合索引覆盖 user_id + 时间范围 |
| 轮次进度 | `idx_user_words_round` | 每轮开始/结束时查询 |
| 熟练度分布 | `idx_user_words_proficiency` | Dashboard 统计用 |
| 答题历史 | `idx_review_logs_user_word` | 单个单词的学习轨迹 |
| 热力图 | `idx_review_logs_created` | 按天分组统计 |
| 学习日历 | `idx_sessions_user` | 用户活跃历史 |

## 6. 字段详解

### 6.1 SM-2 核心字段

`user_words` 表中与 SM-2 算法直接相关的 4 个字段：

```
ef (Easiness Factor)
  ├── 初始值: 2.5
  ├── 范围: [1.3, 3.0]
  ├── 每次复习后更新: ef' = ef + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02))
  └── 含义: 值越大，单词"越容易"，间隔增长越快

interval_days
  ├── 第1次正确 → 1天
  ├── 第2次正确 → 6天
  ├── 第3次+   → round(interval * ef)
  ├── 错误时    → 重置为 1天
  └── 含义: 距离下次复习的天数

repetitions
  ├── 连续正确次数，错误时归零
  └── 含义: 决定间隔计算使用哪个公式

quality (review_logs 中)
  ├── 自动根据正确性 + 响应时间计算
  ├── 0: 错误
  ├── 1-2: 正确但很慢
  ├── 3-4: 正常速度正确
  └── 5: 极快正确（几乎自动反应）
```

### 6.2 熟练度评估字段

`proficiency` 是多维评估的综合结果（0-100），不直接写入 SM-2 计算，仅用于进度展示和轮次判定。

| 维度 | 权重 | 来源字段 |
|------|------|----------|
| 正确率 | 35% | `total_correct / total_attempts` |
| 反应速度 | 25% | `avg_response_time` |
| 连续正确 | 15% | `consecutive_correct` |
| 间隔稳定性 | 15% | review_logs 时间序列 |
| 首次尝试 | 10% | review_logs 首条记录 |

### 6.3 轮次字段

`round` 出现在 `user_words`、`sessions`、`round_completions` 三张表中：

- `user_words.round` — 该学习记录所属的轮次
- `sessions.round` — 该场次所属的轮次
- `round_completions.round` — 已完成的轮次号

**轮次晋级条件**: 当前轮 proficiency ≥ 90 的词数 ≥ 总词数 × 90%

## 7. 连接与配置

### 7.1 数据库初始化

```typescript
import Database from 'better-sqlite3'

export function getDb(): Database.Database {
  const dbPath = process.env.DB_PATH || path.join(process.cwd(), 'data', 'cet4.db')
  const db = new Database(dbPath)
  db.pragma('journal_mode = WAL')     // WAL 模式，读写不互斥
  db.pragma('foreign_keys = ON')      // 启用外键约束
  return db
}
```

### 7.2 PRAGMA 配置

| PRAGMA | 值 | 说明 |
|--------|----|------|
| `journal_mode` | `WAL` | 写前日志，提高并发性能 |
| `foreign_keys` | `ON` | 外键约束（默认 SQLite 关闭） |
| `synchronous` | `NORMAL`（默认） | WAL 模式下 NORMAL 足够安全且速度快 |

### 7.3 Volume 挂载

```yaml
# docker/docker-compose.yml
volumes:
  - ../data:/app/data        # 宿主机 ./data/ → 容器 /app/data/
```

数据库文件在宿主机持久化，容器重启/重建不会丢失数据。

## 8. 常用查询模式

### 8.1 学习相关

```sql
-- 取到期复习词（按紧急度排序）
SELECT w.*, uw.* FROM words w
JOIN user_words uw ON uw.word_id = w.id
WHERE uw.user_id = ?
  AND uw.round = ?
  AND (uw.next_review IS NULL OR uw.next_review <= datetime('now'))
  AND uw.proficiency < 100
ORDER BY uw.next_review ASC
LIMIT ?;

-- 取本轮未学新词（随机模式）
SELECT w.* FROM words w
WHERE w.id NOT IN (
  SELECT uw.word_id FROM user_words uw
  WHERE uw.user_id = ? AND uw.round = ?
)
ORDER BY RANDOM()
LIMIT ?;

-- 取本轮未学新词（按字母顺序）
SELECT w.* FROM words w
WHERE w.id NOT IN (
  SELECT uw.word_id FROM user_words uw
  WHERE uw.user_id = ? AND uw.round = ?
)
ORDER BY
  CASE WHEN substr(upper(w.word), 1, 1) < ? THEN 1
       WHEN substr(upper(w.word), 1, 1) = ? THEN 0
       ELSE 2 END,
  w.word,
  RANDOM()
LIMIT ?;

-- 更新 SM-2 状态
UPDATE user_words SET
  ef = ?, interval_days = ?, repetitions = ?, proficiency = ?,
  next_review = ?, total_correct = ?, total_attempts = ?,
  avg_response_time = ?, consecutive_correct = ?,
  last_reviewed_at = datetime('now')
WHERE user_id = ? AND word_id = ?;
```

### 8.2 统计相关
